# AI-Assisted Terminal

Инструмент для запуска **tmux**-сессии с двумя панелями: слева выполняются команды через обёртку `ai`, справа показывается асинхронный анализ вывода через [NVIDIA NIM / Chat Completions API](https://integrate.api.nvidia.com/). Команды не блокируются ожиданием ответа API.

## Возможности

- Асинхронный вызов API: анализ уходит в фон, терминал остаётся отзывчивым
- Разделение: команды слева, текст анализа справа
- Поддержка мыши в tmux (прокрутка, копирование при наличии `xclip`)
- При каждом запуске установщика обновляются runtime-файлы и блок функции `ai()` в конфиге оболочки
- Ключ API можно передать через переменную окружения или ввести интерактивно (ввод скрыт, в **историю команд не попадает**)

## Требования

- Linux (или WSL с tmux внутри Linux)
- **bash** (установщик и анализатор)
- **tmux** (желательно ≥ 2.6)
- **curl**, **jq**
- **xclip** — опционально; без него привязка буфера обмена в tmux ограничена (на Wayland можно поставить аналог отдельно)
- Ключ [NVIDIA API](https://build.nvidia.com/settings/api-keys)

## Установка зависимостей

```bash
# Ubuntu / Debian
sudo apt update
sudo apt install -y tmux jq curl xclip

# Fedora
sudo dnf install -y tmux jq curl xclip

# CentOS / RHEL (пример)
sudo yum install -y tmux jq curl xclip
```

Сделайте скрипт исполняемым и запустите из каталога репозитория:

```bash
chmod +x setup-ai-tmux.sh
./setup-ai-tmux.sh
```

## Ключ NVIDIA_API_KEY

**Вариант 1 — переменная окружения** (удобно для повторяемых запусков и скриптов):

```bash
export NVIDIA_API_KEY='nvapi-...'
./setup-ai-tmux.sh
```

**Вариант 2 — одной строкой** (ключ виден в окружении процесса, но не обязан попадать в файл истории, если оболочка не логирует env):

```bash
NVIDIA_API_KEY='nvapi-...' ./setup-ai-tmux.sh
```

**Вариант 3** — если переменная **не** задана, установщик запросит ключ **интерактивно**: ввод скрыт (`read -s`), строка **не** является командой оболочки, поэтому **не добавляется в history** так же, как обычный ввод команды.

Без TTY (например, неинтерактивный CI) задайте `NVIDIA_API_KEY` в окружении — иначе скрипт завершится с ошибкой.

## Что делает установщик

1. Проверяет `tmux`, `jq`, `curl`; предупреждает, если нет `xclip`.
2. Создаёт tmux-сессию `ai-workspace` или, если она уже есть, **`ai-workspace-ЧЧММСС`**, без принудительного убийства старой сессии.
3. Пишет/перезаписывает runtime в каталоге **`/tmp/tmux-ai/`** (права на каталог `700`), в том числе:
   - `analyze-command.sh` — запрос к API и вывод результата
   - `left-pane-bashrc`, `right-pane-init.txt` — старт левой/правой панели
4. Добавляет в **`~/.bashrc`** маркированный блок `# >>> tmux-ai function >>>` … `# <<< tmux-ai function <<<` с функцией **`ai()`** и алиасом **`a`**. При каждом запуске установщика блок **заменяется** (старый удаляется, новый вставляется).
5. В **`~/.zshrc`** вставляется **тот же** блок (файл создаётся при необходимости; при каждом запуске блок по маркерам заменяется).
6. Конфиг tmux: создаётся/перезаписывается **`~/.tmux-ai.conf`**. В **`~/.tmux.conf`** добавляется строка `source-file ~/.tmux-ai.conf`, если её ещё не было — **полный `~/.tmux.conf` не затирается**.

После установки подключается к новой сессии: `tmux attach -t <имя_сессии>`.

Чтобы подхватить обновлённую функцию в текущей оболочке:

```bash
source ~/.bashrc    # bash
source ~/.zshrc     # zsh
```

## Использование

В левой панели (или в обычном терминале после `source`):

```text
ai ls -la
ai df -h
a docker ps
```

Функция выполняет команду, печатает её вывод и в фоне запускает анализ (если есть вывод или ненулевой код выхода). Для работы анализа в новых терминалах переменная **`NVIDIA_API_KEY`** должна быть доступна процессу (например, `export` в той же сессии или наследование от родителя).

## Параметры API (в скрипте)

| Переменная / поле | По умолчанию | Описание |
|-------------------|--------------|----------|
| `API_MODEL` | `meta/llama-3.3-70b-instruct` | Модель в `setup-ai-tmux.sh` |
| `API_MAX_TOKENS` | `500` | Лимит токенов ответа |
| `API_TEMPERATURE` | `0.1` | Температура |
| `MAX_OUTPUT_LENGTH` | `2000` | Обрезка вывода команды перед отправкой в API |

Примеры других моделей (актуальный список — в документации NVIDIA):

| Модель | ID |
|--------|-----|
| Llama 3.3 70B | `meta/llama-3.3-70b-instruct` |
| Llama 3.1 8B | `meta/llama-3.1-8b-instruct` |
| Phi-3 Mini | `microsoft/phi-3-mini-4k-instruct` |

## Файлы

| Путь | Назначение |
|------|------------|
| `setup-ai-tmux.sh` | Установщик и генератор конфигов |
| `/tmp/tmux-ai/analyze-command.sh` | Скрипт запроса к API (перезаписывается при каждом запуске установщика) |
| `/tmp/tmux-ai/left-pane-bashrc` | RC левой панели tmux |
| `~/.tmux-ai.conf` | Фрагмент настроек tmux для проекта |
| `~/.tmux.conf` | Пользовательский конфиг; дополняется `source-file` на `~/.tmux-ai.conf` |
| `~/.bashrc` / `~/.zshrc` | Блок с `ai()` (оба обновляются установщиком) |

## tmux: прокрутка

Во многих терминалах прокрутка колёсиком с зажатым **Shift** передаётся tmux; без Shift колесо может скроллить буфер терминала, а не панель tmux.

## Ограничения

- Сложные конструкции shell (пайпы, `&&`, редиректы) в одной строке с `ai` могут вести себя иначе, чем в интерактивной оболочке; для таких случаев запускайте явно: `ai bash -lc 'команда | другая'`.
- Ключ API в переменных окружения виден процессам пользователя; не коммитьте ключ в репозиторий.
